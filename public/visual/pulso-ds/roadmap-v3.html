Roadmap · v3 · Q1–Q2 2027

Pulso DS · Plano v3

A v1 foi esqueleto. A v2 transformou-o em sistema operacional — acessível, falante, governado. A v3 dá ao Pulso identidade própria e linguagem para o que o Crisis Monitor faz de mais singular: assistir o operador em tempo real, em qualquer dispositivo, com IA no centro do fluxo.

Três fases · 14 iniciativas · ~17 semanas se serial; ~12 semanas com paralelismo. Fase α (Identidade) é fundacional — sem ela, β e γ derivam de premissas frouxas. β e γ podem rodar paralelo a partir do fim de α.

North Star da v3

Três premissas que orientam toda decisão de prioridade.

01 · Reconhecível

Pulso sem o roxo ainda é Pulso

Se trocarmos a cor primária amanhã, o sistema deve continuar reconhecível pelo ritmo, tipografia, hairlines e signature shapes. Identidade carregada pela estrutura · não pelo hue.

02 · IA como linguagem

Cris é vocabulário, não componente

A v2 documentou o tom da Cris. A v3 entrega padrões visuais para conteúdo assistido: origem rastreável, confidence, draft com versão, undo, regenerate com restrição.

03 · Paridade móvel

Operador no Uber não é exceção

Crise não espera estação de trabalho. Mobile vira paridade — gestures, offline-first, push handover — e prepara fundação de tokens nativos para quando houver app.

Frentes contínuas
Fora de fase. Rodam paralelo, sem prazo rígido.
contínuo · ergonomia interna
Composição > configuração
Componentes hoje têm props fechadas. Cada caso novo vira RFC. Migrar gradualmente para slot patterns (<Card><Card.Kicker/><Card.Body/></Card>) e primitives compostas reduz fila do council e acelera velocity. Sem big bang · um componente por sprint.
contínuo · escala
i18n real · segundo idioma piloto
Helpers de i18n (v2 · 2.3) existem mas não há segunda língua shipped. Quando produto identificar mercado vizinho (es-419 ou en-US), v3 carrega o piloto · valida densidade tipográfica diferente, plural mais complexo, eventual RTL. Hoje · stand-by.

Premissas que estamos derrubando
Crenças tácitas que travam o sistema na v2.
"Roxo é nossa cor"
→ Cor é ferramenta, identidade é estrutura
Roxo continua. Mas o Pulso precisa ser reconhecível com ele desligado. Fase α inteira é sobre isso · era explicit non-goal da v2 e virou P0 agora.
"Cris é um componente"
→ IA é vocabulário, não chrome
A v2 tratou Cris como assistente (tom, voz). A v3 reconhece que conteúdo gerado por IA precisa de gramática visual própria · origem, confiança, versão, ação assistida. Fase β formaliza.
"Mobile é resgate"
→ Mobile é canal de paridade
A v2 disse: 3 ações no mobile (ack, escalar, comentar). γ vai além · gestures, offline, handover. Não é app nativo ainda, mas é mobile real.
"Componente é a unidade"
→ Pattern e signature shape são unidades também
A v2 promoveu pattern. A v3 promove signature shape · Score Ring, kicker, sentiment bar não são componentes, são identidade. Têm contrato e exigem RFC para mudar.

!
Não-objetivos da v3
O que deliberadamente NÃO entra. Eventualmente v4, ou nunca.
  • × White-label productizado. Arquitetura habilita (v2 · 5.1), mas não há demanda comercial. Volta à mesa quando alguém estiver pagando para isso existir.
  • × App mobile nativo. γ entrega tokens e patterns, não app. Decisão de produto, não de DS.
  • × Cris como produto separado. IA continua embutida no Crisis Monitor. Não há intenção de extrair Cris como SDK público.
  • × Mais componentes novos. v3 amplia ergonomia (composição) e identidade · não breadth. Novos componentes ainda exigem RFC formal.
  • × Refatoração de tokens semantic. A v2 · 5.1 fechou. Mudança nessa camada agora exige RFC duro · sem reorganização cosmética.
