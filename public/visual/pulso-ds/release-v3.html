⚠ Atenção · breaking moderado

O que migra (e o que não)

A v3 mantém retro-compatibilidade total dos tokens da v2. Mudanças visuais ficam em novos tokens e novos componentes · times consumidores migram quando quiserem aproveitar. Apenas duas mudanças exigem atenção:

  • Display headings: --text-display-lg subiu de 28px → 34px e ganhou weight 700. Telas de produto que usam o token diretamente vão crescer ~20% no h1. Avalie caso a caso.
  • Numerais tabulares default: font-variant-numeric: tabular-nums agora é default em .mono e em qualquer elemento dentro de [data-numeric]. Tabelas operacionais ganham coluna mais estável · texto comum em mono pode ter espaçamento ligeiramente diferente.
↻

Como migrar

v2 → v3 · sem big bang

Caminho rápido · 1 dia

Para times que só querem os tokens novos

Atualizar dependência do Pulso DS · rodar codemod do α.3 (cadence tokens) · validar telas críticas. Nada quebra · ganhos visuais aparecem onde os novos tokens são adotados.

  • 1. pnpm up @pulso/tokens-web@^3.0
  • 2. pnpm pulso codemod --version 3.0 · aplica α.3
  • 3. smoke test · 5 telas críticas
  • 4. ship

Caminho completo · 2 sprints

Para times consumindo IA patterns + mobile

Tokens + adoção de AIContentBlock, DraftStack, RecommendationCard, gestures patterns. Backend precisa começar a emitir cris_ref nos eventos auditáveis (β.5). Audit log do produto ganha filtro "Com Cris".

  • Sprint 1: identidade (α) + origin tracking (β.1)
  • Sprint 2: draft & regenerate (β.2) + recommendation (β.4) + audit (β.5)
  • Opcional: mobile (γ) só onde o produto precisa
