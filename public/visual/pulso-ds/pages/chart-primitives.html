Components · Charts · 01

Chart primitives

Seis tipos. Cada um envolvendo Recharts (ou Visx) com tokens Pulso aplicados automaticamente. Wrappers finos, ergonomia consistente, estados loading/error/empty internos. Sem isso, cada tela de monitoramento renderiza um chart diferente de uma biblioteca diferente.

Princípio

Chart é leitura, não decoração. Eixo legível antes de hue elegante. Operador identifica tendência em 1 segundo ou o chart falhou.

Padrões inegociáveis: tokens semânticos, grid pálido (não preto), labels mono tabular-nums, tooltip à distância de hover sem cobrir o ponto, animação que conta a história em vez de roubar o foco. Estados loading/empty/error dentro do componente — não responsabilidade do consumer.

Os seis componentes

Cobertura mínima e suficiente. Compostos (recipes) ficam na 3.5.

01 · LineChart
Score ao longo de 24h · PLS-012
12 pts · 1h step
Score Threshold escalonamento
Quando usar: uma série numérica ao longo do tempo · até 3 séries em comparação. Stroke: 2.25px · linecap round. Endpoint sempre marcado.
02 · BarChart
Menções por tier · últimas 6h
3 categorias
T1 · referência T2 · relevante T3 · eco
Quando usar: comparar quantidades entre categorias (≤ 10). Vertical bars para tempo/ranking; horizontal para listas longas. border-radius: 3 no topo.
03 · AreaChart
Volume de menções · 24h
240 menções
Menções/hora
Quando usar: volume cumulativo · 1 série · ênfase em "quanto". Stack quando > 1 série (com cuidado de leitura). Gradient soft + stroke matching cor.
04 · Sparkline
Mini-chart para metric cards
120 × 28
PLS-012 · Score 67 +12
PLS-014 · Score 42 −8
PLS-018 · Score 38 +0
Quando usar: tendência embutida em texto ou metric card. Sem eixos, sem grid. Cor segue semântica do delta (queda do score = melhora em domínio de crise).
05 · Donut
Sentimento · PLS-012
240 menções
Negativo149
Neutro67
Positivo24
Quando usar: proporção entre 3–5 categorias mutualmente exclusivas. Donut (não pie) — espaço central para a métrica primária. Acima de 5 fatias, use BarChart horizontal.
06 · Heatmap
Cobertura por hora × dia · semana
7 × 24
Intensidade baixo médio alto pico
Quando usar: 2 dimensões discretas + 1 contínua (intensidade). Cells 9–12px com gap 2px. Escala sequencial single-hue da cor principal.

API canônica

Todos os 6 componentes compartilham props · ergonomia idêntica reduz drift entre telas.

// @pulso/charts
// Props compartilhadas por todos os 6 componentes
type CommonChartProps = {
  data: DataPoint[];
  loading?: boolean;          // renderiza skeleton matching
  error?: Error;             // renderiza state de erro
  emptyMessage?: string;      // custom empty copy
  tooltip?: TooltipConfig;     // shape · format · trigger · ver 3.3
  legend?: 'top' | 'bottom' | 'right' | false;
  animation?: 'morph' | 'append' | 'none';
  'aria-label': string;       // obrigatório · alt-text do chart
};

// LineChart
<LineChart
  data={scoreSeries}
  xKey="timestamp"
  yKey="score"
  series={[{ name: "PLS-012", color: "chart-1" }]}
  threshold={{ value: 75, label: "escalonamento" }}
  aria-label="Score de PLS-012 nas últimas 24h"
/>

// Sparkline · sem eixos, sem grid, sem legend
<Sparkline data={values} color="chart-1" width={120} height={28} />

// Heatmap · escala sequencial automática a partir da cor base
<Heatmap
  data={cells}
  xAxis={hours}
  yAxis={days}
  colorScale={["chart-fill-soft", "chart-1"]}
  aria-label="Cobertura por hora e dia"
/>

Estados · loading · empty · error

Cada primitive entrega os três estados internamente. Consumer passa loading, error ou data vazio — chart resolve o resto.

Loading
Skeleton matching · barras horizontais respeitando reduced-motion
Empty
Sem dados na janela
Tente afrouxar o intervalo ou aguardar próxima varredura do radar (≈ 8 min).
Error
Falha ao carregar série
Conexão perdida · retentando em 3s. ref: chart-req-742

Contratos comuns

Decisões que valem para os 6 componentes.

Tipografia
Labels e ticks em var(--font-pulso-mono) · 9–11px · tabular-nums para alinhamento. Nunca sans em valor numérico.
Grid
--chart-grid · linhas horizontais apenas (verticais só em heatmap). Alpha baixo (6%) — sugere, não compete.
Cores
Paleta categórica (chart-1..8) ou semântica (chart-positive/negative/neutral/attention/critical). Detalhe em 3.2 Data Color.
Stroke
Line e área: 2.25px · round caps. Hairlines (separadores no grid): 1px. Endpoint do line: sempre marcado.
Padding
Eixo Y label width: 40px · eixo X label height: 20px · padding interno: 12–16px. Charts pequenos podem reduzir; sparkline elimina.
A11y
aria-label obrigatório · alternativa de tabela disponível via describedBy · ver 1.4 Screen Readers.
Tooltip e legend
Componentes auxiliares dedicados · padrão único · ver 3.3 Axes & auxiliaries.
Animação
Detalhe em 3.4 Real-time motion. Reduced-motion: ver 1.5.

Regras

Faça
  • Use os 6 primitives. Necessidade nova exige RFC, não fork.
  • Passe aria-label em todo chart. Sem isso, falha o build.
  • Use o componente apropriado: linha para tempo, barra para categoria, donut para 3–5 fatias, heatmap para 2D categórica.
  • Confie nos estados internos (loading, error, vazio) — não envolva o chart em wrapper próprio.
  • Ticks numéricos sempre em mono tabular-nums.
Evite
  • Não invente um chart custom porque o existente "não fica bem". Mude o dado, não o componente.
  • Não use Pie acima de 5 fatias. Vire BarChart horizontal.
  • Não use linhas verticais no grid (exceto heatmap). Polui leitura.
  • Não escolha cores fora da paleta. Categórica chart-1..8 ou semântica chart-positive etc.
  • Não anime tudo. Update real-time tem padrão próprio (3.4).
