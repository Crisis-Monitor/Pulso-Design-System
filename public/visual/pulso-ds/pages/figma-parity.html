Tooling · Figma ↔ Code

Figma ↔ Code Parity

Um único JSON é a fonte da verdade. Tokens.studio publica para o Figma · Style Dictionary gera CSS/iOS/Android. Designer e dev divergem em horas, não em semanas.

Por que parity importa

Hoje o token vive em globals.css. Designer abre o Figma e usa cor solta — porque a library não está sincronizada. Em três semanas, "violeta do Pulso" tem três valores: #6C46F5 (CSS), #6D45F6 (Figma estilo) e #6A40F0 (que alguém pintou no quadro). Especificação vira ficção.

A v2 inverte a direção: a fonte é JSON, todo o resto deriva. Designer não escolhe cor — usa estilo da library. Dev não copia hex — importa token. Mudou? Cascateia para os dois lados em um commit.

Fluxo de sincronização

O JSON é o que vai para o git, o que abre PR, o que tem code review. Figma e CSS são consequência.

Schema do JSON

Formato Tokens.studio (DTCG-compatible). Cada token tem $value, $type e referências por {path.to.token}. Grupos por camada → família → step.

// pulso-tokens/tokens.json — fragmento
{
  "primitive": {
    "color": {
      "violet": {
        "500": { "$value": "#6C46F5", "$type": "color" },
        "600": { "$value": "#5532D6", "$type": "color" }
      },
      "gray": {
        "50":  { "$value": "#F5F4FA", "$type": "color" },
        "900": { "$value": "#0F0C1A", "$type": "color" }
      }
    }
  },

  "semantic": {
    "color": {
      "action": {
        "primary":     { "$value": "{primitive.color.violet.500}", "$type": "color" },
        "primary-hover":{ "$value": "{primitive.color.violet.600}", "$type": "color" }
      },
      "surface": {
        "page":     { "$value": "{primitive.color.gray.50}", "$type": "color" },
        "elevated": { "$value": "#FFFFFF", "$type": "color" }
      }
    }
  },

  "component": {
    "button": {
      "primary": {
        "bg":    { "$value": "{semantic.color.action.primary}", "$type": "color" },
        "bg-hover":{ "$value": "{semantic.color.action.primary-hover}", "$type": "color" }
      }
    }
  }
}

Workflow do designer

Editar um token. Da edição ao deploy: 4 passos, ~6 minutos.

Edita no Tokens.studio dentro do Figma
Designer · Figma plugin

Abre o Tokens.studio, navega pra semantic / color / action / primary, ajusta o valor. Vê o impacto imediato em todos os componentes da library.

"Push to GitHub" via plugin
Tokens.studio → branch

Plugin abre um branch tokens/update-action-primary e faz commit no tokens.json. Não toca em nada além disso.

PR: tokens/update-action-primary · +3 lines / -3 lines · reviewers: @ds-lead
CI roda · valida · gera diff
GitHub Actions

Style Dictionary compila. Lint roda contra schema. Diff visual é publicado como comentário do PR (Chromatic) mostrando antes/depois nos componentes afetados.

Merge → publica em cascade
main → npm + Figma

Merge dispara: novo patch de @pulso/tokens no npm, publish da Figma library, post automático no Slack #pulso-changelog.

$ @pulso/tokens@2.4.1 · Figma Pulso DS v2.4.1 · ~6min end-to-end

O que não tem parity

Honestidade: nem tudo casa 1:1. Estes pontos vivem fora do contrato — não são bug, são limites de medium.

Motion
Duração e easing só no código
Figma não tem modelo de animação programática. Especificamos em texto no Figma e referenciamos esta página.
Hover / Press
Variants no Figma, estado no código
Variants representam estados, mas a transição é responsabilidade do componente React.
Charts
Mocks estáticos no Figma
Chart é dado vivo. Figma tem screenshots representativos e tokens de chart-1..8 nas swatches.
Acessibilidade
ARIA é só código
Figma não tem semântica de leitor de tela. Cada componente do library cita seu contrato ARIA na descrição.
