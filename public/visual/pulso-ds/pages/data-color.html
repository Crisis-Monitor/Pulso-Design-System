Escala categórica para dados

Hoje temos 3–4 cores semânticas. Para 5, 8, 12 séries, falta escala categórica perceptualmente equidistante. Esta página fixa a paleta de 8 cores, ordem de prioridade, versões para fundo escuro, simulação color-blind safe e regras de combinação.

Princípio

Cor de dado é função, não estética. Cada hue precisa ser distinguível em: padrão, deuteranopia, protanopia, monocromático. Se uma simulação reduz a paleta a borrão, a paleta perdeu.

Em produto crítico, paleta categórica perceptualmente equidistante é tão importante quanto contraste de texto. Operador daltônico precisa ler 8 séries num multi-line — não pode "confiar na ordem".

Quantas séries · qual cor

Use o mínimo necessário. Chart com 8 séries é menos legível que 4 séries + tabela com o resto.

SériesCores nessa ordemObservação

2chart-1 + chart-2Violet + cyan · contraste máximo entre warm/cool.
3chart-1 + chart-2 + chart-3+ teal. Adiciona dimensão sem ambiguidade.

9+—Não suportado. Use grouping, faceting, ou tabela complementar.

Composição · multi-line

Auditoria · color-blind

A paleta é testada nas 4 condições mais relevantes. Cada strip mostra os 8 hues lado a lado sob filtro. Se você não consegue distinguir as cores, a paleta falhou aquela condição.

Caso-pior · monocromático (impressão B/W · 100% migrânea)
Em monocromático, chart-3 e chart-8 ficam próximos · use os outros 6 primeiro. Em B/W obrigatório (impressão de relatório), adicione padrão (hatch, dotted) além da cor.

Pares perigosos

Algumas combinações são quase indistinguíveis sob daltonismo. Nunca use os dois lado a lado para séries diferentes.

Não · em deuteranopia ficam quase iguais

chart-1 + chart-7 (não usar 7 com poucas séries)

Paleta semântica

Quando o dado tem polaridade (positivo/negativo) ou tier (atenção/crítico), use a paleta semântica. Nunca chart-1..8 para indicar sentido — só para distinguir séries iguais.

Atenção: em Crisis Monitor, queda do score é melhora (verde). Em finance, é o oposto. A semântica é por direção, não por número — sempre escolhida pelo consumer.

Implementação

Tokens
--chart-1 a --chart-8 (categórico) · --chart-positive, -negative, -neutral, -attention, -critical (semântico).
Fills auxiliares
--chart-fill-soft · 14% mix com white (light) / 18% mix com black (dark) · usado em area fills. --chart-fill-strong · 60% / 55% mix · usado em destaques.
Auto-assign
Helper colorForSeries(index) retorna chart-{(index % 8) + 1}. Garante consistência entre charts da mesma tela.
Override por consumer
Possível, mas exige justificativa em PR. Default ganha quase sempre.

Regras

Faça
  • Use chart-1 primeiro · chart-8 só com 8 séries genuinamente necessárias.
  • Para polaridade ou tier, use a paleta semântica · nunca categórica.
  • Em B/W (impressão), some padrão (hatch, dotted) à cor.
  • Em proporção (donut, stacked bar), inclua percentual em texto · não dependa só de hue.
  • Em multi-series, marque endpoint de cada linha com label próprio — operador identifica sem voltar à legenda.
Evite
  • Não combine chart-3 + chart-5 (teal + amber) — colidem em deuteranopia.
  • Não use 9+ séries. Agrupar é solução, não cor extra.
  • Não use cores fora dos tokens. Hex custom não está documentado.
  • Não use vermelho/verde como única forma de comunicar bom/ruim. Sempre acompanhe sinal ou texto.
  • Não inverta a paleta entre charts da mesma tela. chart-1 em um chart, chart-3 em outro para a mesma série quebra o modelo mental.
