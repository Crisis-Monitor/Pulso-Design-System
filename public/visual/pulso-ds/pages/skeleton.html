Skeleton

Placeholder com a forma do conteúdo final. Substitui Spinner em loadings > 1s — preserva layout e reduz percepção de espera.

Primitivos

Cinco shapes — text, title, display, circle, block. Cada um tem altura padrão e raio coerente com o token correspondente.

DISPLAY · 36px · radius-sm
TITLE · 22px · radius-xs
TEXT · 14px · radius 3px
CIRCLE · avatar
BLOCK · arbitrário · radius-md

Padrão · linhas de texto

Para parágrafos, alterne larguras (100% → 100% → 75% → 50%) — cria ritmo natural de quebra de linha.

Padrão · Card de menção

Anatomia composta. Avatar circular, kicker pill, título e linhas de excerto.

Padrão · Métrica

Skeleton que respeita a anatomia do MetricCard — kicker → display → delta.

Padrão · Linhas de tabela

5 linhas é o sweet spot — usuário identifica o padrão sem o skeleton dominar a tela.

Regras

DO
Skeleton deve ter a forma do conteúdo final. Se o card tem avatar circular, o skeleton tem círculo.
DO
Use 3–5 itens de skeleton, não 20. O usuário só precisa entender o padrão.
DO
Anime o shimmer da esquerda para a direita. Em prefers-reduced-motion, mostre estático.
DON'T
Nunca use Skeleton para erros ou estados vazios. Skeleton significa "está vindo".
DON'T
Não troque por Spinner depois — escolha um para a duração toda. Trocar visual no meio do load piora a percepção.
DON'T
Não use Skeleton para load < 300ms. Vai aparecer e sumir, criando flash.
