Localização e formatação

Datas, números, plurais — cada componente precisa de contrato. Crisis Monitor pode escalar para clientes globais; preparar o terreno agora é barato, depois é refator. Esta página fixa o token --locale e os helpers que toda string passa antes de chegar ao operador.

Token --locale

Uma única source of truth. Definido no :root via data-locale no <html> · pode ser sobrescrito por workspace (cliente global) ou por usuário (preferência).

// Resolução de locale — ordem de precedência
1. user.preference        // Settings → Personal → Idioma
2. workspace.locale       // Conta-cliente · default do workspace
3. navigator.language     // Browser hint
4. 'pt-BR'                // Default do produto
Locales suportados v2
pt-BR (default) · en-US. Demais locais ficam em fallback para en-US com warning em console.
Roadmap v3
es-ES · es-MX · fr-FR. Não habilitados sem revisão editorial completa do glossário (2.1) em cada idioma.
Fonte da verdade
Tudo passa por Intl.* nativo + helpers do Pulso. Nunca concatenar string manualmente (${n} mentions).

Helpers · API canônica

Cinco funções cobrem 95% dos casos. Wrap fino sobre Intl que injeta o locale do produto e aplica defaults do Pulso.

// @pulso/i18n
import { formatNumber, formatDate, formatRelative, formatPlural, formatCurrency } from '@pulso/i18n';

// Números — separadores, sinal, casas decimais
formatNumber(1234.5)                          // "1.234,5"   (pt-BR)
formatNumber(1234.5, { signed: true })           // "+1.234,5"
formatNumber(0.84,   { style: 'percent' })       // "84%"
formatNumber(1234567,{ notation: 'compact' })     // "1,2 mi"

// Datas — quatro estilos canônicos
formatDate(date, 'short')                      // "17/05/2026"
formatDate(date, 'long')                       // "17 de maio de 2026"
formatDate(date, 'datetime')                   // "17/05/2026, 14:32"
formatDate(date, 'time')                       // "14:32"

// Relativo — auto-escolhe unidade
formatRelative(-5 * 60_000)                   // "há 5 min"
formatRelative(3 * 3600_000)                  // "em 3 h"

// Plural — escolhe a forma correta por número
formatPlural(42, 'menção', 'menções')        // "42 menções"
formatPlural(1,  'menção', 'menções')        // "1 menção"

// Moeda — symbol, posição, casas decimais
formatCurrency(8400, 'BRL')                  // "R$ 8.400,00"

Sinal explícito
Sempre + e − (U+2212 minus sign, não hífen). Toda delta tem sinal — 0 também tem (+0 com cor neutra).
Fonte mono
Todo número renderiza com var(--font-pulso-mono) + font-variant-numeric: tabular-nums. Coluna de score alinha.
Compact threshold
Switch automático: < 10.000 = forma longa; ≥ 10.000 = compact. Operador escolhe override em Settings.

Posição do sinal · delta

Delta (variação) tem regra firme em todo o produto. Sinal antes do número, mono, sem espaço.

Score sobe
↑ +12    "score subiu 12 pontos · risk-attention"
pulso-mono · tabular-nums
Score cai
↓ −5    Em domínio de crise, queda do score é melhora — cor verde do success
Atenção: invertido em finance
Sem variação
→ +0    Zero é informativo · não é "vazio"
Cor neutra · sem direção
Variação %
+18%    Percent é uma forma do delta · mesmo padrão
Symbol colado ao número

Em crise, queda do score é boa. Token --delta-success aplica-se a melhora, não a "subiu" — direção semântica, não matemática.

Timezone
Display em timezone do operador, sempre indicado quando ambíguo (14:32 BRT, 5:32 PM EST). Toda timestamp persiste em UTC.
Hora 12 vs 24
pt-BR: 24h. en-US: 12h com AM/PM. Não misturar.
Separadores
pt-BR data: / · en-US data: / · ISO: −. Hora: : em todos. Nunca . para hora.

"agora"
Janela de −10s a +10s. Antes ou depois: usa unidade. Operador vê "agora" para o que acabou de chegar.
Cutoff
Acima de 7 dias: cai para short date. Reduz confusão com escalas grandes ("há 47 d" não é útil).
Auto-update
Componente <RelativeTime> atualiza a cada 30s. Em jornada longa, o "há 1 min" vira "há 2 min" sem reload.

Plural · Intl.PluralRules

"1 menção" ≠ "2 menções" ≠ "0 menções". A regra varia por idioma — português tem 2 formas (uma/outra); árabe tem 6. Sempre use o helper.

formatPlural(0,  'menção', 'menções')   // "0 menções"
formatPlural(1,  'menção', 'menções')   // "1 menção"
formatPlural(2,  'menção', 'menções')   // "2 menções"
formatPlural(42, 'menção', 'menções')   // "42 menções"

// Com placeholder em ICU MessageFormat (v3+)
t('mentions', { count }) // {count, plural, one {# menção} other {# menções}}

Direção do texto · RTL

Não suportado na v2. Mas o sistema é preparado: classes utilitárias usam start/end em vez de left/right, e ícones com seta usam logical. Habilitar RTL no futuro é flip de dir="rtl" no <html> + auditoria visual.

Pronto
Spacing utilities (ps-4, me-2), borders (border-s), text-align (text-start).
Auditar antes
Ícones direcionais (chevron, arrow-right), gradientes, sombras laterais, charts.

Regras

Faça
  • Toda string de número/data/plural passa por helper. Zero exceções.
  • Persista timestamp em UTC; renderize no timezone do operador.
  • Use sinal explícito em delta (+12 / −5 / +0). U+2212 para minus, não hífen.
  • Use tabular-nums em colunas numéricas para alinhamento.
  • Em copy nova, pense no plural antes de fixar a estrutura ("1 caso aberto" vs "{N} casos abertos").
Evite
  • Não concatene partes de data: `${day}/${month}/${year}` é proibido.
  • Não escreva pluralização inline: ${n} ${n === 1 ? 'menção' : 'menções'} não cobre 0 e quebra em outros idiomas.
  • Não force ponto decimal em pt-BR ("R$ 8.400.00" é erro de digitação que vira release).
  • Não use timezone implícito. 14:32 em produto global é ambíguo — sempre indique BRT/EST/UTC quando relevante.
  • Não traduza IDs (PLS-012 fica em qualquer locale), termos técnicos (workspace, audit log) ou nomes próprios.
