Foundations · Voice · 02

Microcopy patterns

"Algo deu errado" é hostil. "Oops!" é infantil. Erro de rede, erro de validação, alerta crítico e empty state têm tons distintos — e cada um deles tem uma fórmula canônica. Esta página fixa a voz do produto em cada momento sensível.

Quatro princípios de tom

Cada peça de copy é checada contra estes quatro eixos. Falhar em qualquer um derruba a peça.

01

Direto

O que aconteceu, com qual escopo, o que fazer. Sem rodeios. Operador em pico de crise não tem 3 segundos para parsear sua frase elegante.
"Conexão perdida. Retentando em 3s."
"Houve um problema ao conectar."
02

Calmo

Nem alarmista nem joco. Crise real já carrega adrenalina; a UI segura o tom. Sem exclamações, sem "Atenção!", sem caps.
"Threshold rompido em PLS-012."
"⚠️ ATENÇÃO! Crise crítica!"
03

Sem desculpa

"Desculpe", "infelizmente", "lamentavelmente" não aparecem. Pedem perdão sem oferecer caminho. Substitua por nomeação do estado.
"Não foi possível salvar. Tente em 30s."
"Desculpe, algo deu errado ao salvar."
04

Sem joco

Sem "Ops!", "Eita!", emoji, mascote, exclamação fofa. O produto opera em domínio sensível. Humor irrita quando há perda real.
"Nenhum caso bate com esses filtros."
"Nada por aqui! 🦗"

Comprimento canônico

Cada superfície tem um teto. Acima do teto, o operador não lê.

Erros · quatro tipos

Cada classe de erro tem propósito distinto. O operador precisa saber, em < 1 segundo, se a falha é dele (validation), da rede (network), do sistema (fatal) ou da política (permission).

Erro · Network
Conexão / timeout / 5xx
Falha transiente · provavelmente passa em segundos · ação é esperar ou retry
✗ Hostil
Algo deu errado! Não foi possível processar a requisição. Por favor, tente novamente mais tarde.
✓ Canônico
Conexão perdida Retentando em 3s · automático até reconectar. Tentar agora
Fórmula:  [Estado factual] · [Próximo passo automático] + [CTA opcional manual].  Nunca pede desculpa por algo que vai resolver sozinho.
Erro · Validation
Input inválido
Usuário precisa corrigir · falha local · ação clara e imediata
✗ Vago
Valor inválido Por favor, insira um valor válido.
✓ Específico
Threshold precisa ser entre 0 e 100 Valor atual: −15. Default é 75.
Fórmula:  [Constraint nomeado] · [Valor atual + valor esperado/default].  Nunca "por favor" — operador não está pedindo desculpa, está corrigindo.
Erro · Permission
Não autorizado · 401 / 403
Política do produto · não é bug · operador deve saber a quem pedir
✗ Misterioso
Acesso negado Você não tem permissão para acessar este recurso.
✓ Nomeia o caminho
Apenas analistas declaram crise Encaminhe para o analista de plantão ou peça promoção temporária ao gestor do workspace. Notificar analista
Fórmula:  [Qual papel pode] · [Caminho para escalonar].  Nunca apenas "negado" — sempre nomeia quem é autorizado e como pedir.
Erro · Fatal
Sistema fora · 500 persistente · estado corrompido
Algo está mesmo quebrado · operador precisa de plano B · suporte é envolvido
✗ Sem caminho
Erro inesperado :( Algo de errado não está certo. Recarregue a página e tente novamente.
✓ Plano B nomeado
Caso indisponível agora Persistimos seu rascunho · reabrirá automático quando o sistema voltar. Tempo médio: 4 min. Acompanhe em status.pulso.app. ref: incident-2026-05-17-022
Fórmula:  [Estado] · [O que foi preservado] · [Plano B + onde acompanhar] · [Ref técnica].  Reference number sempre — é o que o operador cola no Slack do suporte.

Empty states · três tipos

Vazio não é falha. Cada tipo tem tom próprio: primeira vez (orientação), após filtro (ajuste), fim do feed (encerramento).

Empty · First time
Workspace ainda sem caso
Onboarding · explica o que vai aparecer aqui
✗ Desculpa
Nada por aqui ainda :( Você ainda não criou nenhum caso. Que tal começar agora? Começar agora
✓ Orientação
Nenhum caso aberto Casos aparecem aqui quando o radar dispara um alerta ou um analista cria manualmente. Conecte fontes em Settings · Fontes para começar a monitorar. Conectar fontes
Empty · No results
Filtro não encontrou
Operador filtrou demais · ação é afrouxar
✗ Inútil
Nenhum resultado encontrado Tente outra busca.
✓ Sugestão concreta
Nenhum caso bate com esses filtros Tente afrouxar o intervalo (atual: últimas 6h) ou remover o filtro de critical. Limpar filtros
Empty · Fim do feed
Estado pretendido · nada pede ação
Vazio aqui é bom · não é problema a resolver
✗ Pessimista
Lista vazia Não há itens para mostrar.
✓ Valida o estado
Nada pede atenção agora 3 casos estão em stable. Próxima varredura do radar em 8 min.

Confirmações destrutivas

Toda ação irreversível tem confirmação. Título nomeia a ação no imperativo; body lista a consequência; CTA repete o verbo. Não usar "Tem certeza?" — vago, infantil, frequentemente respondido sem ler.

Fórmula
Título  =  [Verbo no imperativo]  [Alvo específico]
Body   =  [Consequência concreta].  [O que é preservado].  [Reversibilidade].
CTA   =  [Mesmo verbo do título]
Bom:  "Encerrar PLS-012" · "Todos os operadores perdem acesso ao feed em tempo real. Relatório permanece visível. Não pode ser revertido sem reabrir o caso (audit logged)." · CTA "Encerrar"
Ruim: "Tem certeza?" · "Esta ação não pode ser desfeita." · CTA "OK"

Regras

Faça
  • Nomeie o estado (rede caiu, valor inválido, sistema fora). Sem nome, sem ação.
  • Em erro fatal, sempre incluir request id ou incident ref — é o que o operador cola no chat do suporte.
  • Em CTA, mesmo verbo do título. "Encerrar caso" / "Encerrar". Não "OK".
  • Em destrutiva, nomear o que se perde e o que se preserva. Operador precisa do trade-off completo.
  • Em vazio bom (fim do feed), valide o estado. "Nada pede atenção" é melhor que "Lista vazia".
Evite
  • Não use "Desculpe", "Infelizmente", "Por favor". Pedem perdão sem ajudar.
  • Não use exclamação. Operador em pico não precisa de mais energia na tela.
  • Não use emoji. O produto é institucional.
  • Não use "Algo deu errado", "Tem certeza?", "Erro inesperado". Vagos por construção.
  • Não inicie copy com a marca ("Pulso encontrou…", "O Crisis Monitor diz…"). O operador sabe onde está.
