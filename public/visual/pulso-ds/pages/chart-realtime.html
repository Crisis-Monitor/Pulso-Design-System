Components · Charts · 04

Real-time motion

Crisis Monitor é tempo real · gráfico atualizando mal feito é distrativo, bem feito é informativo. Esta página fixa o easing dedicado, as durações de morph vs append e como o reduced-motion (1.5) entra. Salva a atenção do operador.

Princípio

Animação conta a história. Onde o valor estava · pra onde foi · com que velocidade. Movimento sem narrativa é decoração e rouba foco.

Em chart de monitoramento, três coisas precisam ser animadas: morph de valor existente (linha que sobe) · append de novo ponto (entra do final) · flash de evento de interesse (passou threshold, novo crítico). Tudo o mais é ruído.

Simulação reduced-motion — força transições instantâneas nas demos. Verifique que update permanece (o valor muda) sem o movimento.

Tokens dedicados

Charts usam um easing mais suave que UI normal. Movimento "chart" lembra ponteiro de instrumento · não é emphasized.

TokenCurvaUso
--motion-easing-chartcubic-bezier(0.4, 0.0, 0.2, 1)Easing default de chart · suave em ambas pontas
--motion-duration-chart-morph680msValor existente muda (line/bar/donut update)
--motion-duration-chart-append0msNovo ponto aparece · instantâneo + flash secundário
--motion-duration-chart-flash900msHighlight de novo dado · pulse OU fill flash

Padrão 01 · morph de valor

Valor existente muda. Altura, linha, segmento · interpola entre estado A e B. Easing chart-morph · 680ms.

BarChart · update
Distribuição por tier muda
680ms · easing-chart
O que assistir:  altura interpola, posição-Y interpola. Bar nunca pula. Em reduced-motion, salta direto para o novo valor — informação preservada.

Padrão 02 · append de ponto novo

Nova menção chega · o ponto entra do canto direito. Instantâneo (não anima a entrada) · um pulse de 900ms no novo ponto chama atenção sem cobrir.

LineChart · live point
Score recebe novo data point
append 0ms · pulse 900ms
pontos: 0
Por que append é instantâneo:  se o ponto desliza do lado direito, o operador vê "movimento" onde não há dado novo. Salta para a posição final e pulsa — operador sabe que algo novo chegou sem ser distraído pelo trajeto inexistente.

Padrão 03 · flash em célula nova

Heatmap recebe atualização em célula específica · célula pisca da cor de destaque para a cor final. Informa "essa célula é nova" sem animar o grid inteiro.

Heatmap · cell update
Cobertura por hora · célula nova
flash 900ms
O que assistir:  a célula "agora" pisca chart-1 forte e estabiliza em chart-fill-soft. Resto do grid não se move.

Reduced motion · contrato

Sob prefers-reduced-motion: reduce, charts cortam transição mas mantêm update. Operador vê o novo valor instantaneamente · sem perder informação.

Morph
Cai para 0.001ms · valor "salta" para o novo · flash secundário opcional garante percepção da mudança.
Append
Já era instantâneo · pulse 900ms é cortado · color flash 1-frame substitui (cor brilha por 1 tick, ≈ 16ms).
Flash
Pisca permanece (é informação, não decoração) · 1 ciclo · 1.6s · classe .keep-anim.
Skeleton
Shimmer cai para placeholder estático em var(--muted). Ver 1.5.

Implementação · API

// Hook que une matchMedia + token de duração
function useChartMotion(type: 'morph' | 'append' | 'flash') {
  const reduced = usePrefersReducedMotion();
  if (reduced) {
    return { duration: 0.001, easing: 'linear' };
  }
  switch (type) {
    case 'morph':  return { duration: 680,  easing: 'var(--motion-easing-chart)' };
    case 'append': return { duration: 0,    easing: 'linear' };
    case 'flash':  return { duration: 900,  easing: 'var(--motion-easing-chart)' };
  }
}

Regras

Faça
  • Use morph 680ms para valor existente · append 0ms+pulse para novo ponto.
  • Use var(--motion-easing-chart), não --motion-easing-emphasized. Chart não é entrada/saída de modal.
  • Em reduced-motion, mantenha o update · corte a transição · mantenha flash (informação).
  • Threshold rompido sempre flash + anúncio polite.
  • Initial render via skeleton · jamais "fly-in" das séries.
Evite
  • Não anime initial render. Skeleton resolve.
  • Não anime o ponto novo entrando da lateral. Salta + pulsa.
  • Não use easing emphasized em chart. É excesso de personalidade.
  • Não acumule animações concorrentes. Se há flash, não há morph simultâneo · cancela um e prioriza o evento mais novo.
  • Não use animação para chamar atenção (piscar em loop). Flash 1 ciclo basta · loop irrita.
