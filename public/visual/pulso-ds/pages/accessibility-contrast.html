Política do Pulso

O DS adota WCAG 2.1 AA como mínimo para todo texto de produto. AAA é o alvo para texto crítico (alertas, mensagens de erro).

NívelCritérioQuando aplicar
AAAContraste ≥ 7.0 : 1 (texto normal) · ≥ 4.5 (texto grande)Texto crítico: alertas, confirmação destrutiva, erros
AAContraste ≥ 4.5 : 1 (texto normal) · ≥ 3.0 (texto grande)Default obrigatório para todo texto de produto
AA LargeContraste ≥ 3.0 : 1 — só vale para texto ≥ 18pt (24px) ou ≥ 14pt (18.5px) boldDisplay, kickers, badges visuais — nunca body
FAILContraste < 3.0 : 1Proibido para qualquer texto, exceto disabled e decorativo

Regras de uso

Faça
  • Use fg-strong, fg ou fg-secondary em qualquer body text — todos passam AAA.
  • Use fg-muted em labels e captions sobre card; em surface-2 só para texto ≥14px.
  • Use fg-subtle em placeholders, timestamps e metadados — agora passa AA.
  • Audite combinações novas usando esta página antes de subir PR.
Evite
  • Não use fg-ghost em texto. É decorativo (separadores, watermarks).
  • Não use fg-disabled em qualquer texto que não seja em controle disabled.
  • Não invente um cinza intermediário — use o token mais próximo.
  • Não confie em cor sozinha para sinalizar risco — combine com ícone + label (ver redundância na iniciativa 1.2).
