Reduced motion

Animação em monitoramento real-time pode disparar enxaqueca, ataque vestibular ou ansiedade. Respeitar prefers-reduced-motion: reduce é trivial, barato e elimina uma classe inteira de incidentes. O Pulso desliga o movimento decorativo e mantém o movimento que informa.

Simulação reduced-motion — força prefers-reduced-motion: reduce nesta página. As demos abaixo refletem o estado escolhido.

Quem se beneficia

Não é população nicho. Dor-de-cabeça antes de pico de crise é exatamente quem precisa do Crisis Monitor funcionando.

~35%

Sensibilidade vestibular

Vertigem desencadeada por parallax, slide-in agressivo, blur progressivo. Inclui pacientes com migrânea vestibular, doença de Ménière, pós-AVC.

~15%

Migrânea / aura

Pulso luminoso, flash, transição entre cores saturadas pode disparar crise. Operador com aura ativa precisa baixar estímulo agora — não em 3 cliques.

+

Ansiedade · TDAH · fadiga

Movimento atrai a atenção. Em jornada longa de monitoramento, animação decorativa rouba foco que deveria estar no sinal real.

Princípio

Movimento que decora some. Movimento que informa permanece.

Hover lift, page transition, modal slide-up, micro-bounce — tudo decoração. Pulse de "live", spinner de loading, progress bar — informação. Cortar a primeira lista é grátis; cortar a segunda equivale a esconder o sinal vital do paciente.

O contrato global · globals.css

Uma única regra no fim de globals.css neutraliza animação e transição em todos os elementos. Componentes específicos que merecem manter o movimento usam a classe .keep-anim como opt-in explícito.

/* globals.css — base sweep */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.001ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.001ms !important;
    scroll-behavior: auto !important;
  }

  /* Opt-in: keep informational motion alive */
  .keep-anim, .keep-anim * {
    animation-duration: 1.6s !important;
  }
}
0.001ms (não 0)
Mantém o evento transitionend disparando — código JS que espera transição terminar não trava. Truque clássico vindo da Adobe.
!important
Único lugar onde aceitamos !important no Pulso. A user preference vence qualquer cascade.
.keep-anim
Classe explícita. Aplica-se a containers de pulse, progress, spinner e qualquer movimento que carregue informação.

Hook JS · usePrefersReducedMotion

Para animações controladas via JS (charts, sprites, GSAP/Framer Motion), consultar a preferência diretamente. CSS sozinho não cobre canvas / WebGL / requestAnimationFrame.

// hooks/usePrefersReducedMotion.ts
export function usePrefersReducedMotion() {
  const [reduced, setReduced] = useState(
    () => window.matchMedia('(prefers-reduced-motion: reduce)').matches
  );
  useEffect(() => {
    const mq = window.matchMedia('(prefers-reduced-motion: reduce)');
    const onChange = (e) => setReduced(e.matches);
    mq.addEventListener('change', onChange);
    return () => mq.removeEventListener('change', onChange);
  }, []);
  return reduced;
}

// Uso em chart de tempo real
const reduced = usePrefersReducedMotion();
<LineChart data={series} animationDuration={reduced ? 0 : 600} />

Como ativar no sistema

Antes de PR, todo dev/designer testa nas três plataformas. O toggle no topo desta página é simulação — não substitui o flag real do OS.

macOS

  1. Ajustes do Sistema
  2. Acessibilidade → Tela
  3. Marcar Reduzir movimento

Windows 11

  1. Configurações
  2. Acessibilidade → Efeitos visuais
  3. Desligar Efeitos de animação

iOS / Android

  1. iOS: Ajustes → Acessibilidade → Movimento → Reduzir movimento
  2. Android: Configurações → Acessibilidade → Remover animações

Regras

Faça
  • Use a regra global em globals.css. Não tente "esconder" animação componente a componente.
  • Marque com .keep-anim apenas o que é informação (pulse, spinner, progress).
  • Em modal/sheet, mantenha o fade de opacidade, corte o translate. Orientação espacial fica.
  • Em JS-driven animation, leia matchMedia('(prefers-reduced-motion: reduce)'). CSS não cobre canvas.
  • Teste antes do PR ativando o flag no OS, não só simulando no DevTools.
Evite
  • Não corte animações de carregamento. Spinner imóvel = sistema travado.
  • Não use display:none para "remover" animação — quebra layout. Use animation-duration: 0.001ms.
  • Não dependa de animação para hierarquia. Se o usuário não vê o movimento, ainda deve entender a estrutura.
  • Não adicione confete, parallax e overshoot no modo normal "porque é bonito". Eles não estão no DS.
