01
Metric · Sparkline · Delta
TrendCard
Métrica primária com tendência inline. O cartão default do dashboard de casos · resume o estado num cartão de 280×140.
usado em: Dashboard · Radar
componentes: Sparkline
SCORE · PLS-012
67
+12 · 6h
−6hagora
Quando usarCard de metric primária com tendência. Ideal para 4–6 lado a lado em dashboard. Não substituir por chart maior — TrendCard é "leitura de canto-de-olho".
ComposiçãoKicker mono · valor display mono tabular-nums · delta chip (cor por direção semântica) · sparkline 240×50 · X labels em mono.
<TrendCard
  label="Score · PLS-012"
  value={67}
  delta={{ value: 12, window: "6h" }}
  series={scoreSeries}
/>
02
Line · Threshold · Tooltip
ScoreOverTime
Score do caso ao longo de janela, com linha de threshold de escalonamento. O chart canônico da aba Resumo.
usado em: Crisis Detail · Resumo
componentes: LineChart + ChartTooltip
Quando usarAba Resumo de Crisis Detail · trends de risco. Single-series sempre (não comparar casos aqui). Threshold sempre marcado quando configurado.
ComposiçãoLineChart · cor chart-1 · threshold em risk-attention dashed · endpoint marker + label · tooltip ao hover.
<ScoreOverTime
  series={scoreSeries}
  threshold={{ value: 75 }}
  windowHours={24}
/>
03
Area · Annotation · Velocity
MentionVolume
Volume agregado com anotação em picos. Operador identifica quando algo explodiu sem precisar passar mouse.
usado em: Cobertura
componentes: AreaChart
Quando usarCobertura agregada. Quando há pico notável, ancore com pin · cor do pin sinaliza nível (attention/critical).
ComposiçãoAreaChart · gradient soft · pin opcional via annotations={[]}. Pin texto curto, max 24 chars.
<MentionVolume
  series={volumeSeries}
  annotations={[{
    at: "17:00", label: "Pico · 38 menções", level: "critical"
  }]}
/>
04
Heatmap · Sequential scale
SentimentHeatmap
Densidade de menções negativas por dia × hora. Identifica padrão temporal — quando dói mais? Operacionalmente útil para planejar plantão.
usado em: Cobertura · Padrão
componentes: Heatmap
Quando usarQuando o pattern temporal importa para decisão (alocar plantão, antecipar pico). Sempre acompanhe com leitura textual do pico abaixo do chart.
ComposiçãoHeatmap · escala sequencial single-hue de chart-1 · leitura textual obrigatória ("Pico em X").
<SentimentHeatmap
  data={cells}
  scale="sequential-violet"
  summary="Pico: Qua 14h–16h · 4× média"
/>
05
Donut · Legend com contagem
TierBreakdown
Distribuição de menções por tier de veículo. Mostra de onde está vindo a pressão — T1 (Folha, Globo) preocupa mais que T3 (perfis pequenos).
usado em: Cobertura · Briefing
componentes: Donut
T1 · referência92
T2 · relevante76
T3 · eco54
Outros18
Quando usarDistribuição em 3–4 categorias. Sempre legenda com contagem absoluta + cor. Mais de 5 fatias: virar BarChart horizontal.
ComposiçãoDonut · total no centro mono · legenda à direita com count tabular-nums alinhado.
<TierBreakdown
  total={240}
  segments={[
    { name: "T1", count: 92 },
    { name: "T2", count: 76 },
    { name: "T3", count: 54 },
    { name: "Outros", count: 18 }
  ]}
/>

08
Multi-line · Comparação
CompareCases
Score de 2–4 casos em paralelo. Apoia decisão de priorização ("qual escalar primeiro?"). Aba dedicada do produto.
usado em: Compare
componentes: LineChart · endpoint labels
Quando usarComparação direta · 2 a 4 casos. Acima de 4: virar tabela com sparkline.
ComposiçãoMulti-line · cores 1–4 na ordem · endpoint label inline (não depende de legenda externa).
<CompareCases
  caseIds={["PLS-012", "PLS-014", "PLS-018"]}
  windowHours={24}
/>
09
Donut parcial · Taxa
VelocityGauge
Taxa de crescimento (menções/hora) vs threshold de aceleração. Mostra se está acelerando, estável ou desacelerando.
usado em: Radar · Briefing
componentes: Donut parcial
0%+100%
menções/h cresceu 72% nas últimas 3h vs janela anterior
Quando usarQuando taxa de crescimento é a métrica primária da decisão. Verde < 25% · âmbar 25–60% · vermelho > 60%.
ComposiçãoDonut parcial 180° · cor por tier semântico (success/attention/critical) · valor central + label semântico.
<VelocityGauge
  growthPct={72}
  window="3h"
  thresholds={{ warn: 25, danger: 60 }}
/>
10
Timeline · Pins de evento
CoverageTimeline
Timeline horizontal com pins de incidentes externos sobreposta a área de menções. Conecta o evento real ao espelhamento na cobertura.
usado em: Histórico · Relatório
componentes: AreaChart + pins
Vazamento detectado
Pronunciamento oficial
Apuração concluída
Quando usarHistórico longo + 2–5 eventos-chave do caso. Pin cor por nível do evento. Texto curto, max 28 chars.
ComposiçãoAreaChart com baseline · pins sob o baseline · labels acima com leader-line implícita (mesma cor).
<CoverageTimeline
  series={dailyVolume}
  pins={[
    { at: "2026-05-18", label: "Vazamento detectado", level: "critical" },
    { at: "2026-05-19", label: "Pronunciamento oficial", level: "attention" },
    { at: "2026-05-20", label: "Apuração concluída", level: "monitor" }
  ]}
/>

Regras

Faça
  • Use a recipe nomeada · não recombine primitives ad-hoc na tela.
  • Mantenha o componente · só passe data + props · não inverta cores ou eixos.
  • Em qualquer recipe com heatmap ou top-N, escreva o "summary" textual abaixo do chart (pico, líder, gargalo).
  • Em recipes com pin/anotação, mantenha o texto curto (≤ 28 chars).
Evite
  • Não crie uma recipe nova sem RFC. O custo de fragmentação é alto.
  • Não acumule recipes na mesma viewport · ≤ 3 charts por tela.
  • Não compare casos via TrendCards · use CompareCases.
  • Não use SentimentHeatmap para mostrar contagem absoluta — é densidade.
