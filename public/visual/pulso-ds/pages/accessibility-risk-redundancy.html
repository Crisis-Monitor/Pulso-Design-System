Redundância de Risco

Em produto de gestão de crise, cor sozinha é proibida para sinalizar risco. Operador daltônico (1 em 12 homens, 1 em 200 mulheres) precisa distinguir attention de high sem ler hue — mesmo em pico de adrenalina, lendo de canto-de-olho. Este sistema impõe glifo + label + cor, sempre.

Por que isso importa

Só cor — proibido
Acima · paleta normal · Abaixo · simulação grayscale (≈ deuteranopia severa). Os tons ficam quase iguais.
Glifo + cor + label — obrigatório
Stable Monitor Attention High Critical
Mesmo sem cor, o operador distingue os 5 níveis pela forma e pelo texto. Lê-se em qualquer condição.

Sistema de glifos

Cinco formas distintas, escolhidas para máxima diferenciação geométrica. Densidade de fill cresce com o nível — quanto pior a crise, mais "pesado" o glifo.

stable
Círculo cheio · base calma
monitor
Círculo vazio · observando
attention
Meio cheio · meia-elevação
high
Losango · escalado
critical
Quadrado cheio · travado

Tamanhos canônicos

12px em chips inline · 14px em metadata · 20px em score rings · 32px em hero. Stroke 1.5 para outline shapes, fill sólido nos demais.

API e ARIA

O componente <RiskGlyph> recebe level e renderiza glifo decorativo (aria-hidden). A semântica vai no componente pai (RiskChip, RiskRail).

// React API canônica
<RiskChip level="attention" label="Attention" />
<RiskRail level="critical" id="PLS-012" title="Vazamento de dados — APP" />
<RiskRing level="attention" score={62} />

// Renderização (RiskChip)
<span role="status" aria-label="Risco attention">
  <RiskGlyph level="attention" aria-hidden="true" />
  <span>Attention</span>
</span>

Guardrail · ESLint rule (proposta)

Para tornar o anti-pattern bloqueante, não só documentado, vamos publicar uma regra ESLint que detecta uso de --risk-* sem componente Pulso correspondente.

// .eslintrc.js — proposta v2
{
  rules: {
    'pulso/risk-redundancy': ['error', {
      // CSS classes com --risk-* exigem ícone irmão
      checkClasses: ['risk-*', 'bg-risk-*', 'text-risk-*'],
      requireSiblings: ['RiskGlyph', 'aria-label']
    }]
  }
}

Plugin eslint-plugin-pulso a publicar junto com a v2.0.0. Owner técnico definido durante a fase 5 (Governança).

Regras

Faça
  • Sempre componha glifo + label + cor em qualquer indicador de risco.
  • Use os componentes Pulso (RiskChip, RiskRing, RiskRail) — eles já trazem o glifo correto.
  • Em UI sem texto (chip muito pequeno), use aria-label="Risco attention".
  • Mantenha a forma do glifo idêntica em qualquer tamanho (12 a 32px). Não invente um glifo "menor".
Evite
  • Não use bg-risk-critical sozinho num <div> custom — adicione o glifo ou volte para o componente Pulso.
  • Não substitua um glifo por emoji ou ícone Lucide. O sistema é fechado: 5 formas, sem extensão.
  • Não use os mesmos glifos para significar outras coisas (severidade técnica, prioridade de tarefa). São reservados para risk.
  • Não dependa de animação para diferenciar níveis (piscar mais rápido em critical etc). Animação reforça, nunca substitui.
