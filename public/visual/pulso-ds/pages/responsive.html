Responsive contract

Crisis Monitor é desktop-first · mas operador no Uber abre pelo celular. Sem padrão, mobile vira "tem que existir" sem direção. Mobile não é paridade · é resgate: o operador chega na crise, executa 3 ações e segue para o desktop quando puder.

Princípio

Mobile entrega o crítico em 3 toques. Desktop é a ferramenta operacional completa. Tablet é desktop responsivo, não mobile dilatado.

Desktop primeiro porque é onde operador passa 95% do tempo. Mobile não é reduced-feature desktop — é uma jornada diferente: alerta chega → operador entende → ack/escala/comenta → segue para o desktop em 30 min. Filtros, analytics, comparativos? Ficam fora.

Mobile · 3 ações que existem

Operador no Uber tem 4 minutos e uma mão. Mobile entrega 3 ações · nada mais. Filtros, analytics, comparativos e configurações ficam ocultos sob "⋯ mais".

01

Ack

Reconhecer o alerta para parar de receber push. Single tap · não exige reflexão. Reverte com undo 8s.

02

Escalar

Subir para o próximo tier. Triggers automáticos de notificação ao analista de plantão. Não exige texto · justificativa pode ser adicionada depois.

03

Comentar

Texto curto. "Estou a caminho do escritório". Vai para o log do caso · todos veem.

Container queries > media queries

Sempre que possível, o componente decide layout por tamanho do container, não por viewport. Card de métrica colapsa baseado no espaço que tem · não em "é tablet".

/* ✓ component-level reflow · independente da viewport */
.metric-card {
  container-type: inline-size;
}
@container (width < 200px) {
  .metric-card .delta { display: none; }
}

/* ✗ apenas viewport-based · ignorra layouts compostos */
@media (max-width: 640px) {
  .metric-card .delta { display: none; }
}

Viewport queries ficam reservadas para decisões de navegação (sidebar/topbar/FAB) e composição global (1-col vs 3-col). Tudo dentro de um card decide pelo container.

Regras

Faça
  • Desktop primeiro. Mobile é jornada paralela · não versão reduzida.
  • Mobile entrega 3 ações: ack · escalar · comentar. Tudo o mais fica em "⋯ mais".
  • Use container queries para decisões internas de componente · viewport queries só para chrome global.
  • Em mobile, hit target ≥ 44px sempre (mesmo em compact).
  • Audit log e Compare são desktop-only · banner amigável no mobile.
Evite
  • Não tente paridade. Mobile tentando ser desktop fica pior em ambos.
  • Não use hover em mobile. Substitua por tap · long-press · ou revele inline.
  • Não use modal centralizado em mobile. Bottom sheet sempre.
  • Não esconda navegação primária em mobile. Bottom tabs ou hamburger persistente.
  • Não use scroll horizontal exceto em tabelas com indicadores claros.
