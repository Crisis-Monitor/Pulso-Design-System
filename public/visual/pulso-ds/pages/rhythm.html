Hairlines, ritmo e densidade calibrada

A α.1 mostrou que hairlines passam o teste sem cor, mas perdem força em demasia. A v3 separa borda como sinal de contexto de borda como decoração — e introduz tokens de cadência para que o ritmo vertical seja decisão, não acidente.

Princípio · regra de ouro

Hairline existe para separar contexto. Quando o espaço já o faz, a borda é decoração — e decoração compete com sinal.

Crisis Monitor é cockpit. Cada elemento estrutural que não serve a um operador acelerado vira ruído. A v3 corta hairlines decorativas — não para minimalismo, mas para devolver atenção ao dado.

Cadência vertical · 3 tokens

v3 nomeia o ritmo entre blocos. Cada token tem uma intenção · onde aplicar e quando trocar.

tight
--rhythm-cadence-tight
6px (0.375rem)
Itens da mesma unidade. Kicker → label → valor. Rows próximas em lista densa. Onde os elementos são partes do mesmo "objeto" mental.
beat
--rhythm-cadence-beat
14px (0.875rem)
Sub-blocos do mesmo contexto. Sub-secções de um card · pares title+body. Onde os elementos compõem o mesmo card, mas são unidades semanticamente distintas.
breath
--rhythm-cadence-breath
26px (1.625rem)
Mudança de contexto. Entre seções da página · entre card-de-caso e tabela-de-cobertura. Espaço como "respiração", marca passagem mental para outro assunto.

Cadência por densidade

A v2 · 4.5 entregou densidade como preferência. v3 entrega cadência calibrada · os tokens encolhem/crescem proporcionalmente, mantendo o sistema reconhecível em qualquer modo.

Token Compact Comfortable · default Spacious
--rhythm-cadence-tight 4px 6px 8px
--rhythm-cadence-beat 10px 14px 20px
--rhythm-cadence-breath 18px 26px 36px
--density-pad-card 10px 12px 14px 16px 20px 22px
--density-pad-page 32px 40px 56px 64px 80px 88px

Proporções fixas · um modo é versão escalada do outro, não uma redefinição. Operador percebe densidade como "ritmo mais apertado" · não como "tela diferente".

Anti-patterns que a v3 trava

Comportamentos comuns em DSs que envelhecem mal. v3 os bloqueia via lint, code review e Figma constraint.

✕ Anti-pattern
Box dentro de box dentro de box
Card → border interna → sub-card → border interna. 3 níveis de "ah, isto é separado". Visual ruim, hierarquia ambígua.
✓ Pulso v3
1 container + cadência interna
Um card externo. Hairlines apenas onde mudança de contexto pede. Espaço/kicker fazem o resto.
✕ Anti-pattern
Border-left colorida sem razão
Sticker colorido no canto · decora, não informa. Vira o "Bootstrap alert" genérico.
✓ Pulso v3
Border-left = signature
Border-left 3px reservada para evidência citada · marca conteúdo de origem externa. Promovido a signature shape em α.4.
✕ Anti-pattern
Divisores em toda lista
Hairline 1px entre todos os itens · ruído quando o espaço interno do row já é suficiente.
✓ Pulso v3
Divisor apenas quando o row é compacto
Row com padding ≥ 16px · sem divisor. Row compacto (densidade compact) · border-faint ajuda olho.
✕ Anti-pattern
Border + shadow + bg distinto
Tudo ao mesmo tempo. Modal com sombra + borda + bg branco sobre página clara · sinalização redundante.
✓ Pulso v3
Pick one · sombra ou borda
Em superfícies flutuantes · sombra. Em superfícies cosídas à página · borda. Não ambos.

Regras formalizadas · v3

1 · Uma borda por nível de profundidade

Nenhum elemento dentro de um card pode ter borda completa. Sub-blocos usam hairline-top apenas onde mudança de contexto pede · nunca borda integral.

2 · Padding ≥ 16px? Sem divisor entre rows

Quando o ar interno do row já dá respiração visual, divisor é ruído. Compact (10px) precisa de divisor · comfortable (14px+) não.

3 · Section ≠ Card

Seção da página é semântica de leitura · não container visual. Sem borda, sem fundo distinto. Apenas kicker + título + cadência breath.

4 · Border-left 3px = evidência

Reservada. Não usar como decoração de card, alerta, sidebar item, kicker. Promovida a signature shape em α.4.

5 · Sombra ou borda em superfícies flutuantes

Modal, sheet, popover, tooltip · escolhe uma. Default · sombra. Borda volta só em high-contrast (a11y).
