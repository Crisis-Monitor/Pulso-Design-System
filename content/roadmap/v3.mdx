# Pulso DS · Plano v3

> **Status:** Concluída · 25 mai 2026 · registro histórico do plano

Este plano foi entregue — ver [Release v3.0](/release/v3) para o consolidado do que foi publicado.

A v1 foi **esqueleto**. A v2 transformou-o em **sistema operacional** — acessível, falante, governado. A v3 dá ao Pulso **identidade própria** e **linguagem para o que o Crisis Monitor faz de mais singular**: assistir o operador em tempo real, em qualquer dispositivo, com IA no centro do fluxo.

Três fases · 14 iniciativas · 12 semanas com paralelismo β ‖ γ.

---

## North Star da v3

### 01 · Reconhecível — Pulso sem o roxo ainda é Pulso

Se trocarmos a cor primária amanhã, o sistema deve continuar reconhecível pelo ritmo, tipografia, hairlines e signature shapes. Identidade carregada pela estrutura — não pelo hue.

### 02 · IA como linguagem — Cris é vocabulário, não componente

A v2 documentou o tom da Cris. A v3 entrega padrões visuais para conteúdo assistido: origem rastreável, confidence, draft com versão, undo, regenerate com restrição, audit trail.

### 03 · Paridade móvel — Operador no Uber não é exceção

Crise não espera estação de trabalho. Mobile vira paridade — gestures, offline-first, push handover — e prepara fundação de tokens nativos para quando houver app.

### β.4 · Recommendation & action `P1`

## Fase γ · Mobile como paridade

> **4 iniciativas · ≈ 6 semanas · P1/P2 · paralela a β**
>
> Operador no Uber não é exceção. Crise não espera estação de trabalho.

### γ.1 · Mobile-first patterns base `P1`

5 gestures canônicos com feedback háptico · hit-target adaptativo · zona segura do polegar mapeada.

**Entregáveis:**
- 5 gestos: swipe-to-ack · pull-to-refresh · long-press · drag bottom-sheet · two-finger select
- Hit-target adaptativo: 44px default, 48px crítico, 32px dismiss
- Bottom-anchored actions · zona segura do polegar
- Página **Patterns · Mobile · Gestures**

_Esforço: 5d · Owner: design · Deps: α.3_

### γ.2 · Offline-first contract `P1`

Tabela de capacidades por estado de rede · queue lifecycle · 4 níveis de freshness · schema `queued_action`.

**Entregáveis:**
- Tabela de capacidades: 9 ações classificadas em online/instável/offline
- UI de queue · ações pendentes visíveis, retry transparente
- Indicador de freshness em 4 níveis (fresh/stale/warning/offline)
- Schema `queued_action` com idempotency key
- Página **Patterns · Mobile · Offline**

_Esforço: 6d · Owner: eng + design · Deps: γ.1_

### γ.3 · Push handover web ↔ mobile `P1`

Deep-link schema · sync < 2s entre dispositivos · 3 tiers de push com SLA mensurável.

**Entregáveis:**
- Deep-link schema · `pulso://crisis/{id}?action=ack`
- State sync em < 2s entre dispositivos do mesmo usuário
- 3 tiers: crítico (vibra + som, bypass DND) · alto · monitor (silent)
- SLA: 99.5% entrega tier 01 em < 10s
- Página **Patterns · Mobile · Handover**

_Esforço: 4d · Owner: eng · Deps: γ.1_

### γ.4 · Native token foundation `P2`

Pipeline 1 JSON → 3 outputs · parity matrix 100% · drift detector em CI · sample apps minimalistas.

**Entregáveis:**
- iOS sample app consumindo `@pulso/tokens-ios` em SwiftUI
- Android sample app consumindo `@pulso/tokens-android` em Compose
- Audit de drift web ↔ nativo · 100% paridade nos primitives
- Drift detector em CI bloqueia merge com discrepância
- Documentação · onboarding para time nativo futuro
- Página **Foundations · Native Token Foundation**

_Esforço: 4d · Owner: eng · Deps: v2 · 5.2_

## Não-objetivos da v3

- **App mobile nativo.** γ entrega tokens e patterns, não app. Decisão de produto, não de DS.
- **Cris como produto separado.** IA continua embutida no Crisis Monitor. Não há intenção de extrair Cris como SDK público.
- **White-label productizado.** Arquitetura habilita (v2 · 5.1), mas não há demanda comercial.
- **Mais componentes novos.** v3 amplia ergonomia (composição) e identidade · não breadth.
- **Refatoração de tokens semantic.** A v2 · 5.1 fechou. Mudança nessa camada agora exige RFC duro.
