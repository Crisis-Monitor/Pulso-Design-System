# Dialog

> **Organism**
>
> Dialog interrompe o fluxo. Use só quando a tarefa exige atenção exclusiva ou decisão explícita.

Dialog é uma janela modal centralizada para confirmações, formulários curtos e revelações focadas. Diferente de `Sheet`, ele bloqueia o contexto por trás e pede que o usuário resolva a tarefa antes de voltar.

Se a pessoa precisa comparar com a lista, revisar contexto lateral ou continuar explorando, use [`Sheet`](/components/atoms/sheet) ou [`Responsive Sheet`](/components/organisms/responsive-sheet). Se a ação é destrutiva ou irreversível, use `Alert Dialog`.

```tsx
import * as React from "react"

type PulsoDialogProps = {
  open: boolean
  title: string
  description: string
  children: React.ReactNode
  primaryAction: React.ReactNode
  secondaryAction?: React.ReactNode
  onOpenChange: (open: boolean) => void
  closeLabel?: string
  closeOnEscape?: boolean
}

export function PulsoDialog({
  open,
  title,
  description,
  children,
  primaryAction,
  secondaryAction,
  onOpenChange,
  closeLabel = "Fechar",
  closeOnEscape = true,
}: PulsoDialogProps) {
  const titleId = React.useId()
  const descriptionId = React.useId()
  const panelRef = React.useRef<HTMLDivElement>(null)

  React.useEffect(() => {
    if (!open) return

    const previousActiveElement = document.activeElement as HTMLElement | null

    function onKeyDown(event: KeyboardEvent) {
      if (event.key === "Escape" && closeOnEscape) {
        onOpenChange(false)
      }
    }

    document.addEventListener("keydown", onKeyDown)
    panelRef.current?.focus()

    return () => {
      document.removeEventListener("keydown", onKeyDown)
      previousActiveElement?.focus()
    }
  }, [closeOnEscape, onOpenChange, open])

  if (!open) return null

  return (
    <div
      role="presentation"
      className="fixed inset-0 z-50 grid place-items-center bg-black/45 p-4"
      onMouseDown={(event) => {
        if (event.target === event.currentTarget) onOpenChange(false)
      }}
    >
      <div
        ref={panelRef}
        role="dialog"
        aria-modal="true"
        aria-labelledby={titleId}
        aria-describedby={descriptionId}
        tabIndex={-1}
        className="w-full max-w-lg rounded-[var(--radius-lg)] border border-[var(--border)] bg-[var(--popover)] shadow-[var(--shadow-lg)] outline-none"
      >
        <div className="flex items-start justify-between gap-4 border-b border-[var(--border)] p-5">
          <div>
            <h2 id={titleId} className="text-lg font-semibold text-[var(--fg)]">
              {title}
            </h2>
            <p id={descriptionId} className="mt-1 text-sm leading-6 text-[var(--fg-muted)]">
              {description}
            </p>
          </div>

          <button
            type="button"
            aria-label={closeLabel}
            onClick={() => onOpenChange(false)}
            className="rounded-[var(--radius-sm)] px-2 py-1 text-[var(--fg-muted)] hover:bg-[var(--muted)] hover:text-[var(--fg)] focus-visible:outline focus-visible:outline-[var(--ring-width)] focus-visible:outline-offset-2 focus-visible:outline-[var(--ring)]"
          >
            ×
          </button>
        </div>

        <div className="p-5 text-sm text-[var(--fg)]">{children}</div>

        <div className="flex items-center justify-end gap-2 border-t border-[var(--border)] p-4">
          {secondaryAction}
          {primaryAction}
        </div>
      </div>
    </div>
  )
}
```

### Exemplo de uso

```tsx
export function ConfirmDispatchDialog() {
  const [open, setOpen] = React.useState(false)

  return (
    <>
      <button
        type="button"
        onClick={() => setOpen(true)}
        className="rounded-[var(--radius-md)] bg-[var(--primary)] px-4 py-2 text-sm font-semibold text-[var(--primary-fg)]"
      >
        Revisar despacho
      </button>

      <PulsoDialog
        open={open}
        onOpenChange={setOpen}
        title="Confirmar despacho assistido"
        description="Revise a recomendação da Cris antes de enviar. A decisão final continua humana e auditável."
        secondaryAction={
          <button className="rounded-[var(--radius-md)] border border-[var(--border)] px-4 py-2 text-sm font-medium text-[var(--fg)]">
            Voltar
          </button>
        }
        primaryAction={
          <button className="rounded-[var(--radius-md)] bg-[var(--primary)] px-4 py-2 text-sm font-semibold text-[var(--primary-fg)]">
            Confirmar despacho
          </button>
        }
      >
        <p>Origem: relatório reputacional validado há 2 min.</p>
        <p className="mt-2 text-[var(--fg-muted)]">
          Esta ação registra trilha de auditoria com usuário, horário e fonte.
        </p>
      </PulsoDialog>
    </>
  )
}
```

## Quando usar

- Confirmações de consequência média.
- Formulários curtos, com até quatro campos.
- Revelação focada que precisa de leitura exclusiva.
- Decisão que precisa pausar o fluxo antes de continuar.

Não use Dialog para drill-down, filtros avançados, navegação lateral ou conteúdo longo. Nesses casos, use Sheet/Responsive Sheet ou página dedicada.

## Dialog vs Sheet vs Alert Dialog

| Superfície | Uso | Regra |
|---|---|---|
| `Dialog` | Foco curto e decisão bloqueante | Interrompe o fluxo |
| `Sheet` | Drill-down e edição contextual | Preserva contexto lateral |
| `Responsive Sheet` | Mesmo fluxo em desktop/mobile | Adapta side/bottom sheet |
| `Alert Dialog` | Ação destrutiva ou irreversível | Nomeia consequência e exige confirmação explícita |

## Variantes

| Variante | Uso | Regra |
|---|---|---|
| Informacional | Mensagem focada com ação simples | Evite se um Alert inline resolve |
| Confirmação | Continuar/cancelar com consequência clara | Descrição deve explicar impacto |
| Formulário curto | Até quatro campos | Mais que isso vira Sheet ou página |
| Largo com evidências | Lista curta de evidências ou preview | Não transforme em dashboard |

## Anatomia

| Parte | Papel |
|---|---|
| Kicker opcional | Contextualiza tipo de decisão |
| Title | Nomeia a decisão ou conteúdo |
| Description | Explica consequência, limite ou incerteza |
| Body | Conteúdo curto e focado |
| Footer | Cancelar à esquerda, ação primária à direita |
| Close | Disponível quando a saída é segura |

## Acessibilidade

- Focus trap dentro do Dialog.
- `Esc` fecha apenas quando não houver perda de dados ou ação crítica pendente.
- Focus retorna ao trigger ao fechar.
- Título e descrição precisam estar associados ao modal.
- Não use apenas cor para comunicar severidade.

## Regras

1. Dialog bloqueia; não use para conteúdo exploratório.
2. Formulários longos vão para Sheet ou página dedicada.
3. Ação primária precisa nomear o verbo real, não “OK”.
4. Descrição deve nomear consequência quando houver risco.
5. Ação destrutiva ou irreversível usa `Alert Dialog`.
6. Não abra Dialog sobre Dialog sem decisão explícita de stack.
7. Conteúdo assistido por IA precisa indicar base e incerteza.
8. Fechar não pode executar ação nem descartar dado sem confirmação.
