# Page Header

> **Molecule**
>
> Page Header ancora a leitura de uma superfície. Ele diz onde o operador está, qual decisão a página suporta e quais ações são seguras naquele contexto.

Use Page Header em dashboards, detalhes de crise, listas operacionais, páginas de configuração e telas de relatório. Ele deve evitar título genérico: o cabeçalho precisa nomear escopo, estado e consequência.

```tsx
type PulsoPageHeaderProps = {
  eyebrow?: string
  title: string
  description?: string
  meta?: React.ReactNode
  primaryAction?: React.ReactNode
  secondaryAction?: React.ReactNode
  children?: React.ReactNode
}

export function PulsoPageHeader({
  eyebrow,
  title,
  description,
  meta,
  primaryAction,
  secondaryAction,
  children,
}: PulsoPageHeaderProps) {
  return (
    <header className="border-b border-[var(--border)] pb-6">
      <div className="grid gap-4 lg:grid-cols-[minmax(0,1fr)_auto] lg:items-start">
        <div className="min-w-0">
          {eyebrow ? (
            <p className="font-mono text-xs font-semibold uppercase tracking-[var(--tracking-kicker)] text-[var(--fg-muted)]">
              {eyebrow}
            </p>
          ) : null}

          <h1 className="mt-2 text-3xl font-bold tracking-tight text-[var(--fg)]">
            {title}
          </h1>

          {description ? (
            <p className="mt-3 max-w-3xl text-base leading-7 text-[var(--fg-muted)]">
              {description}
            </p>
          ) : null}

          {meta ? <div className="mt-4">{meta}</div> : null}
        </div>

        {(primaryAction || secondaryAction) ? (
          <div className="flex flex-wrap gap-2 lg:justify-end">
            {secondaryAction}
            {primaryAction}
          </div>
        ) : null}
      </div>

      {children ? <div className="mt-6">{children}</div> : null}
    </header>
  )
}
```

## Quando usar

- Início de página com escopo operacional claro.
- Superfícies que têm ação primária ou filtros abaixo.
- Páginas de decisão, revisão, configuração ou relatório.
- Contextos em que atualização, fonte ou status precisam aparecer antes do conteúdo.

Não use Page Header para seções internas curtas; prefira um heading simples com descrição local.

## Anatomia

| Parte | Papel | Regra |
|---|---|---|
| Eyebrow | Localização e categoria | Use mono/kicker; não substitui breadcrumb |
| Title | Nome da superfície | Específico, curto e orientado à tarefa |
| Description | Escopo e consequência | Explique o que a página permite decidir |
| Meta | Freshness, fonte, status ou escopo | Use quando impacta confiança operacional |
| Actions | Comandos principais | Uma ação primária; secundárias com menor peso |
| Children | Slot opcional | Use para KPIs, tabs ou filtros imediatamente relacionados |

## Conteúdo

- O título deve nomear o objeto: `Casos em monitoramento`, não `Dashboard`.
- A descrição deve explicar consequência: o que muda se o operador agir.
- Evite prometer automação quando a decisão final é humana.
- Quando houver IA, explicite origem, revisão ou trilha auditável.
- Se houver atraso de dados, mostre freshness no `meta`.

## Ações

1. Use no máximo uma ação primária.
2. Ação destrutiva não deve ser a ação primária do cabeçalho.
3. Botões lado a lado devem compartilhar altura e alinhamento de baseline.
4. Ação primária deve estar relacionada ao escopo da página inteira.
5. Se a ação depende de seleção, deixe-a no toolbar ou bulk action, não no Page Header.

## Relação com outros padrões

- Use `Filter Bar` abaixo quando filtros alteram o conjunto de dados.
- Use `Tabs` no slot `children` quando a página tem visões irmãs do mesmo objeto.
- Use `Metric Card` abaixo quando os KPIs resumem o mesmo escopo do título.
- Use `AI Audit Trail` quando o cabeçalho introduz conteúdo assistido pela Cris.

## Regras

1. Não use título decorativo ou marketing em superfície operacional.
2. Não esconda estado crítico apenas em badges abaixo da dobra.
3. Não duplique breadcrumb no eyebrow.
4. Não coloque múltiplas ações concorrentes com peso primário.
5. Não use descrição como release note; ela precisa apoiar a decisão da página.
