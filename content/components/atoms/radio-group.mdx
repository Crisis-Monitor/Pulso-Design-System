# Radio Group

> **Atom / selection primitive**
>
> Radio é escolha exclusiva: exatamente uma opção do grupo.

Use Radio Group quando todas as opções precisam estar visíveis para comparação e o conjunto é pequeno. Se houver muitas opções, hierarquia complexa ou busca, use Select.

## Import

```tsx
import { Label } from "@/components/ui/label"
import { RadioGroup, RadioGroupItem } from "@/components/ui/radio-group"
```

## Uso básico

```tsx
<RadioGroup defaultValue="daily">
  <div className="flex items-center space-x-2">
    <RadioGroupItem value="daily" id="daily" />
    <Label htmlFor="daily">Diário</Label>
  </div>
  <div className="flex items-center space-x-2">
    <RadioGroupItem value="weekly" id="weekly" />
    <Label htmlFor="weekly">Semanal</Label>
  </div>
</RadioGroup>
```

## Quando usar

- 2–5 opções mutuamente exclusivas.
- Quando todas as opções precisam ficar visíveis.
- Configurações e preferências.
- Escolhas com consequência operacional clara.
- Segmented control para opções curtas e frequentes.

Não use Radio para múltipla escolha; use Checkbox. Não use para on/off simples; use Switch.

## Variantes

| Variante | Uso | Regra |
|---|---|---|
| Standard | Configurações e formulários | Label sempre associado |
| Com descrição | Opções com consequência | Explique impacto operacional |
| Card | 2–4 escolhas com peso | Facilita comparação |
| Segmented | 2–4 opções curtas e frequentes | Bom para período, modo, escala |

## Radio vs Select vs Checkbox

| Componente | Uso |
|---|---|
| Radio | Uma escolha entre poucas opções visíveis |
| Select | Muitas opções ou opções com hierarquia |
| Checkbox | Múltipla escolha ou boolean independente |
| Switch | Liga/desliga imediato |

## Acessibilidade

- Navegação por setas dentro do grupo.
- Label sempre associado via `htmlFor`.
- Estado selecionado precisa ser programático, não apenas visual.
- Descrição deve ser lida junto da opção quando informa consequência.

## Regras

1. Use Radio para 2–5 opções.
2. Mais de 5 opções tende a Select.
3. Card variant deve ter poucas opções e descrições curtas.
4. Segmented é Radio com apresentação compacta, não Button Group genérico.
5. Não use cor como único indicador de seleção.
6. Cada opção precisa explicar consequência quando afeta operação.
