# Conceito

## O que é o Pulso

Pulso é o sistema de design do **Orchids Crisis Manager** — uma plataforma de gestão de crise e monitoramento de reputação digital.

O nome "Pulso" vem da ideia central do produto: **manter o pulso da situação**. Em crise, cada segundo conta e a interface precisa transmitir informação clara e imediatamente.

## Por que existe

Antes do Pulso, cada tela do Orchids era construída de forma independente:

- Cores diferentes para o mesmo status
- Botões com estilos inconsistentes
- Espaçamento arbitrário
- Sem padrão de acessibilidade
- Dificuldade para criar novas features

O Pulso resolve isso com **tokens, componentes e padrões compartilhados**.

## Para quem

- **Desenvolvedores** — componentes prontos, tokens documentados, imports diretos
- **Designers** — linguagem visual consistente, decisões registradas
- **Produto** — velocidade de entrega, consistência de UX
- **Usuários finais** — interface previsível, acessível, profissional

## Filosofia

> "Em crise, a interface não pode ser a crise."

O Pulso segue 5 princípios:

1. **Clareza sobre ornamento** — cada pixel existe para comunicar
2. **Ação sobre decoração** — ferramenta, não vitrine
3. **Dados primeiro** — o conteúdo dita o layout
4. **Status visível** — o usuário sempre sabe o estado do sistema
5. **Consistência contextual** — mesmo padrão, mesma expectativa

## Arquitetura

Pulso é um sistema de **atomic design adaptado**:

```
Foundations → Átomos → Moléculas → Organismos → Padrões → Templates
```

Cada nível compõe o anterior. Tokens fundamentam átomos. Átomos compõem moléculas. E assim por diante.
