# Pulso Design System

Pulso é o sistema de design do **Orchids Crisis Manager** — plataforma de gestão de crise, monitoramento de reputação e decisão assistida.

## v3 em uma frase

A v3 dá identidade própria ao sistema: **Pulso sem o roxo ainda é Pulso**. A interface passa a depender menos de cor e mais de tipografia, ritmo, signature shapes, marcação de IA e contratos mobile.

## O que o sistema garante

- **Clareza operacional** para estado, risco, prioridade e consequência.
- **Consistência visual** entre produto, documentação e templates.
- **IA auditável**: conteúdo da Cris tem origem, confiança, fontes e trilha.
- **Paridade mobile**: gestos, offline e hit targets críticos documentados.
- **Evolução incremental**: tokens v3 são majoritariamente aditivos.

## Estrutura

| Nível | Descrição | Exemplos |

| **Foundations** | Tokens, tipografia, cores, espaçamento, ícones e motion | Base de todo o sistema |
| **Componentes** | Átomos, moléculas e organismos | Button, Badge, MetricCard, Table |
| **Padrões** | Composições para cenários reais | Risk Levels, Crisis Status |
| **Templates** | Estruturas completas de página | Dashboard, Crisis Detail, Settings |
| **Conceitual** | Filosofia, princípios e regras de IA | Conceito, padrões, AI Rules |
| **Release** | Narrativa e migração | v3.0 |

## Princípios rápidos

1. **Clareza sobre ornamento** — cada elemento comunica algo verificável.
2. **Estado antes de estética** — risco, origem e consequência vêm primeiro.
3. **Dados primeiro** — números críticos usam mono e alinhamento tabular.
4. **IA não é chrome** — Cris aparece como vocabulário, origem e trilha.
5. **Mobile não é exceção** — ações críticas respeitam paridade e toque.
